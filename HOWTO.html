1 — Which manifest is which

There are two manifest files in extension/. Never edit or swap them manually — the build scripts handle everything.

FileTargetUsed by
extension/manifest.json Firefox desktop & Android npm run dev, npm run build, npm run lint
extension/manifest.chrome.json Chrome desktop npm run build:chrome only — never touched by any other command
How the Chrome build works npm run build:chrome temporarily copies manifest.chrome.json over manifest.json, runs the build, then immediately restores the Firefox manifest — even if the build fails. After the command returns, manifest.json is always the Firefox version.

The only code difference between the two manifests is:

  • Firefox manifest has browser_specific_settings (gecko ID + min version for Android)
  • Chrome manifest has minimum_chrome_version instead, no gecko settings

All logic lives in content.js and is shared between both browsers via a one-line shim at the top of the file (const _b = typeof browser !== 'undefined' ? browser : chrome).

2 — Building

CommandOutput fileUpload this to
npm run build dist/fallen_london_companion-0.3.zip Firefox Add-ons (AMO) FF only
npm run build:chrome dist/fallen_london_companion-chrome.zip Chrome Web Store Chrome only
Before any release build: bump the version Update "version" in both extension/manifest.json and extension/manifest.chrome.json to the same string (e.g. "0.4"), then run both build commands.

3 — Testing on Firefox desktop

Option A — Dev mode (recommended for active development)

cmd /c "npm run dev"

Opens a dedicated Firefox window on fallenlondon.com with the extension loaded. Any file save in extension/ auto-reloads the extension. No reinstall needed.

Option B — Temporary install from folder

  1. Open about:debugging#/runtime/this-firefox
  2. Click Load Temporary Add-on…
  3. Navigate to extension/ and select manifest.json
  4. Visit https://www.fallenlondon.com
Updating after a code change Do not use the Reload button — it doesn't pick up all changes. Always Remove the extension, then Load Temporary Add-on… again. Temporary installs are cleared when Firefox closes.

Option C — Test the actual release zip

  1. Run npm run build → produces dist/fallen_london_companion-0.3.zip
  2. In Firefox: Add-ons and themes → gear icon → Install Add-on From File…
  3. Select the zip from dist/

This installs as a permanent (unsigned) add-on. Unsigned add-ons only work in Firefox Developer Edition or Nightly, or if xpinstall.signatures.required is set to false in about:config.

4 — Testing on Firefox Android

One-time phone setup

  1. Install Firefox Nightly on your Android device (required for unsigned extension installs)
  2. Open Firefox Nightly → tap the menu (⋮) → Settings → About Firefox Nightly
  3. Tap the Firefox logo 5 times rapidly — you'll see "Debug menu enabled"
  4. Go back to Settings — a new "Install extension from file" option now appears

Installing / updating the extension

  1. Run npm run build on your PC → produces dist/fallen_london_companion-0.3.zip
  2. Transfer the zip to your phone (email, Google Drive, USB — anything works)
  3. On the phone: Settings → Install extension from file → select the zip
  4. Tap Add on the install prompt
  5. Open https://www.fallenlondon.com in Firefox Nightly and log in
Re-installing after an update Firefox Android does not auto-update sideloaded extensions. To update: go to Add-ons (Settings → Add-ons), remove the old version, then install the new zip via the same flow above.

5 — Testing on Chrome desktop

Option A — Load unpacked (recommended for development)

  1. Open chrome://extensions
  2. Enable Developer mode (toggle in the top-right corner)
  3. Click Load unpacked
  4. Select the extension/ folder (not a zip — the folder itself)
  5. Visit https://www.fallenlondon.com

To reload after a code change: click the refresh icon on the extension card in chrome://extensions, then hard-reload the game tab.

Option B — Test the actual release zip

  1. Run npm run build:chrome → produces dist/fallen_london_companion-chrome.zip
  2. In chrome://extensions (Developer mode on): drag the zip onto the page, or use Pack extension if you want a .crx
Do not load the Chrome zip in Firefox, or the Firefox zip in Chrome They will install but behave incorrectly (wrong gecko/chrome API assumptions in the manifest).

6 — Publishing to Firefox Add-ons (AMO)

File to upload: dist/fallen_london_companion-0.3.zip Built by npm run build. This is the Firefox zip — it contains manifest.json with gecko settings.
  1. Bump the version in extension/manifest.json and run npm run build
  2. Go to addons.mozilla.org/en-US/developers/addons
  3. Click your extension → Upload New Version
  4. Upload dist/fallen_london_companion-0.3.zip
  5. When asked "Is this add-on listed on AMO?":
    • Choose "On your own" to get a signed .xpi for self-distribution (no review, instant)
    • Choose "On AMO" for public listing (requires review, takes days)
  6. If self-distributing: download the signed .xpi — this is what installs cleanly on Android without needing the debug menu
Source code submission AMO may ask for source code if the extension is minified or obfuscated. This extension is plain unminified JS — you can submit the same zip.

7 — Publishing to Chrome Web Store

File to upload: dist/fallen_london_companion-chrome.zip Built by npm run build:chrome. This is the Chrome zip — it contains the chrome manifest (no gecko settings).
  1. Bump the version in extension/manifest.chrome.json (and manifest.json to keep them in sync) and run npm run build:chrome
  2. Go to chrome.google.com/webstore/devconsole
  3. Click New Item (first time) or select your existing item → Package → Upload new package
  4. Upload dist/fallen_london_companion-chrome.zip
  5. Fill in the Store Listing (description, screenshots, category)
  6. Submit for review — Chrome review typically takes 1–3 days for updates
One-time developer account fee Chrome Web Store requires a $5 one-time developer registration fee if you haven't paid it already.

8 — Release checklist

  1. Run node run-tests.js — all tests pass
  2. Run cmd /c "npm run test:all" — unit + Playwright tests pass
  3. Bump version in both extension/manifest.json and extension/manifest.chrome.json
  4. Run npm run build → verify dist/fallen_london_companion-0.3.zip exists
  5. Run npm run build:chrome → verify dist/fallen_london_companion-chrome.zip exists
  6. Test Firefox zip via temporary install (Option B in §3) — check core features on desktop
  7. Test on Firefox Android (§4) — primary target, test all features
  8. Test Chrome zip via Load unpacked or zip drag-in (§5) — check core features
  9. Commit and push: git add -A && git commit -m "vX.Y" && git push
  10. Upload fallen_london_companion-0.3.zip FF only → AMO (§6)
  11. Upload fallen_london_companion-chrome.zip Chrome only → Chrome Web Store (§7)
Never cross-upload The Firefox zip has gecko-specific settings that will cause a Chrome submission error (and vice versa). The filenames make it unambiguous: -chrome.zip goes to Chrome, the other goes to AMO.
